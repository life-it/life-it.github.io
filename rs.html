Резервирование подключения к сети интернет

Для некоторых клиентов критический важно иметь постоянный доступ в интернет.

Существует 2 вида потребности:

-подключаться надо вам к интернету: бесперебойный доступ к интернету без привязки к выделенным IP адресам клиента извне;
-подключаться надо из интернета к вам: бесперебойный доступ из интернета к оборудованию клиента с привязкой к выделенным IP адресам.


Для первого случая достаточно приобрести оборудование с поддержкой данного функционала.

Для второго случая существует 2 варианта реализации.


Первый: при подключении нескольких провайдеров, для каждого из них необходимо оплатить нужное количество IP адресов.

При данном типе подключения важно чтобы сервисы которые будут подключаться к IP адресам клиента имели возможность переподключаться к доступным IP адресам клиента.

Например: некоторые провайдеры IP телефонии требуют от клиента адрес их сервера телефонии, при переподключении на резервного провайдера интернета, необходимо будет обращаться к провайдеру IP телефонии для смены привязанного IP адреса, причем как для перехода на резерв так и при возврате на основной канал.


Второй: аренда собственного пула IP адресов и настройка протокола обмена трафиком BGP с провайдером.

При таком варианте от провайдера покупается только ширина канала. Единственное ограничение, то что сам провайдер должен иметь возможность оказания данной услуги.


Дополнительные сведения:

Цена настройки и обслуживание протокола BGP указанна для default view и дистрибутива PfSense.
